Что это?

CheatCraft - это программный обеспечение для вписывания в память и чтение ее. Любая игра это - набор разменных, которые выполняют определенную функцию: отображает количество хп, патронов или блоков... И в каждой переменной есть свой адрес в памяти. Зная его, умея пользоваться CheatCraft и имея небольшие навыки пользоваться компьютером ты можешь получить преимущества в играх.
Так что же какие навыки нам будут полезны? Хоть немного понимать что такое идентификатор процесса, адресов в памяти и умение пользоваться cmd(cd, dir, cls будет достаточно).
Ну а если ты знаешь С ++ я для тебя оставил Исходный код, ты можешь пидлаштувать под себя и скомпилирувать в новую программу)

Стоит ли пользоваться этим?

Конечно нет, играй честно, но раз ты зашел сюда, то попробуй.
В чем преимущество над другими читами? Это легкость использования и скорость. тебе не придется путаться в кнопках, а просто в терминале изменить значение и нажать Enter.
Как много недостатков? Это онечно малый функционал... Ты не сможешь летать или видит врага через стены а лишь прибавить себе хп. А также ты должен сам искать адресоа в памяти например через Cheat Engine. Скоро будет включен парсер памяти, но он работает довольно медленно(хотя и удобно).

Если есть какие-то идеи на щот него, то переходи по ссылке "Author" и напиши свои пожелания)

Instruction

Здесь ты узнаешь как пользоваться CheatCraft.
Если ты правильно установил и настроил программу то после ее запуска ты увидишь полный перечень команд.

  • hack process

    Сначала попробуй ввести "hack process" для вывода всех процессов и показа их идентификатора. С этим ты узнаешь process_id нужной тебе программы. Это самая простая, но одна из самых важных команд.

  • hack *process_id* in *address*

    Перейдем к более сложной "hack *process_id* in *address*", где process_id это идентификатор процесса, а address адрес в памяти. Где нам взять адрес? Ты можешь воспользоваться программой Cheat Engine. Ней довольно легко пользоваться и легко найти туториалы. Эта команда выведет тебе значение переменной по этому адресу.

  • hack *process_id* where *address* = *value*

    Это то же команда, но там ты читали из памяти, а здесь вводим новое значение (*value*).
    hack 6459 where 0x7ffc6ff8b834 = 100

Как это работает?

Это программное обеспечение написано на С++ с использованием WinAPI. давайте Розберемся как оно работает!
Cначала включил необходимые модули для работы программы: 

 
#include <windows.h> 
#include <tlhelp32.h>

Работает это таким образом, что мы читаем и списываем в память данные с помощью функции ReadProcessMemory(...) для чтения и WriteProcessMemory(...) для записи.

ReadProcessMemory: 

 
BOOL ReadProcessMemory( 
  HANDLE  hProcess,
  LPCVOID lpBaseAddress,
  LPVOID  lpBuffer,
  SIZE_T  nSize,
  SIZE_T  *lpNumberOfBytesRead
);

WriteProcessMemory: 

 
BOOL WriteProcessMemory( 
    HANDLE hProcess,
    LPVOID lpBaseAddress,
    LPCVOID lpBuffer,
    SIZE_T nSize,
    SIZE_T* lpNumberOfBytesWritten
  );

Поочередно найдем нужные аргументы. Так, для начала у нас HANDLE hProcess:


  HANDLE hProcess ; 
  hProcess = OpenProcess(PROCESS_ALL_ACCESS, 0, ProcessId); // ReadProcessMemory
  hProcess = OpenProcess((0x0008) | (0x0020) | (0x0400), 0, ProcessId); // WriteProcessMemory

Дальее LPVOID lpBaseAddress - это идентификатор процесса(Process id). его можно найти используя FindWindowW или написать функцию самом:


  DWORD FindProcessId (LPCTSTR ProcessName)            
  {
      PROCESSENTRY32 pt;
      HANDLE hsnap = CreateToolhelp32Snapshot (TH32CS_SNAPPROCESS, 0);
      pt.dwSize = sizeof (PROCESSENTRY32);
      if  ( Process32First (hsnap, &pt)) { 
        do  {
            if  (!lstrcmpi (pt.szExeFile, ProcessName)) {
                CloseHandle (hsnap);
                return  pt.th32ProcessID;
              }
          } while  (Process32Next(hsnap, &pt));
      }
      
      std::cout << "Process not found (return 0)" << std::endl;
      CloseHandle (hsnap); 
      return 0;
  }

Мы дошли до адресу LPVOID lpBaseAddress. Это объект, указывающий на определенные данные, но не хранящий их. Например:


DWORD Address = 0x22fe00; // (LPVOID)Address

Осталось только указать буфер и его размер. Буфер должен иметь тип как и у элемента, который вы читаете из памяти; размер буфера указывает с помощью sizeof(buffer)
